# RAGRO Architecture — Dependency Injection

## Why Dependency Injection?

Without DI (Dependency Injection), to create a `LoginBloc` you would need to write:

```dart
// Without DI — tedious, fragile, and impossible to test:
final prefs = await SharedPreferences.getInstance();
final dio = Dio();
final apiClient = ApiClient(dio);
final localDs = AuthLocalDataSource(prefs);
final remoteDs = AuthRemoteDataSource(apiClient);
final repo = AuthRepositoryImpl(remoteDs, localDs, apiClient);
final loginUser = LoginUser(repo);
final loginBloc = LoginBloc(loginUser);
```

With DI, it is simply:

```dart
// With DI — simple, decoupled, 100% replaceable in tests:
final loginBloc = getIt<LoginBloc>();
```

`get_it` resolves the entire dependency chain automatically, because each class declares what it needs via its constructor and `injectable` generates the registration code.

---

## get_it as a Service Locator

RAGRO uses the `get_it` package as a global service locator. A single access point (`getIt`) holds all registered instances in the app.

```dart
// lib/core/di/injection.dart
final getIt = GetIt.instance;

@InjectableInit()
Future<void> configureDependencies() async => getIt.init();
```

The `configureDependencies()` function is called in `main.dart` before `runApp()`.

## injectable Annotations

### `@lazySingleton`

Creates a single instance that is reused throughout the application. The instance is created only the first time it is requested (lazy). Use for:
- Services, Repositories, DataSources (shared state or single connection)
- `AuthBloc` (global authentication state)
- `AppRouter` (single router)
- `ApiClient`, `Dio` (one HTTP connection per app)

```dart
@lazySingleton
class AuthRemoteDataSource { ... }

@lazySingleton
class LoginUser { ... }

@lazySingleton
class AuthBloc extends Bloc<AuthEvent, AuthState> { ... }
```

### `@injectable`

Creates a new instance on every call to `getIt<T>()`. Use for BLoCs that **should not** be global — each page receives its own clean instance.

```dart
@injectable
class LoginBloc extends Bloc<LoginEvent, LoginState> { ... }

@injectable
class RegisterBloc extends Bloc<RegisterEvent, RegisterState> { ... }

@injectable
class HomeBloc extends Bloc<HomeEvent, HomeState> { ... }
```

> **Rule**: `AuthBloc` is `@lazySingleton` (global state). All other BLoCs are `@injectable` (per-page state).

### `@LazySingleton(as: Interface)`

Registers the concrete implementation mapped to its interface. This is how dependency inversion works in practice: the BLoC depends on `AuthRepository` (interface), and get_it resolves `AuthRepositoryImpl` (implementation).

```dart
@LazySingleton(as: AuthRepository)
class AuthRepositoryImpl implements AuthRepository { ... }

@LazySingleton(as: ConsumerProfileRepository)
class ConsumerProfileRepositoryImpl implements ConsumerProfileRepository { ... }
```

## The Dependency Chain

When `getIt<LoginBloc>()` is called, get_it resolves the entire chain automatically:

```
getIt<LoginBloc>()
    └── LoginUser (lazySingleton)
            └── AuthRepository (interface → resolves AuthRepositoryImpl)
                    └── AuthRepositoryImpl (lazySingleton)
                            ├── AuthRemoteDataSource (lazySingleton)
                            │       └── ApiClient (lazySingleton)
                            │               └── Dio (lazySingleton)
                            ├── AuthLocalDataSource (lazySingleton)
                            │       └── SharedPreferences (factoryAsync — resolved at boot)
                            └── ApiClient (same singleton already created)
```

All of this resolution is **automatic** — the `injection.config.dart` file generated by `injectable` contains the registration code in the correct order.

---

## Where to Find the Generated File

```
lib/core/di/injection.config.dart   ← GENERATED — do not edit manually
```

This file is generated by `build_runner` and should be committed to the repository so that builds work without needing to run the generator on every machine.

---

## How to Regenerate After Changes

Whenever you add a new class with the `@injectable`, `@lazySingleton`, or `@LazySingleton(as: ...)` annotation, run:

```bash
dart run build_runner build --delete-conflicting-outputs
```

For watch mode (regenerates automatically on each save):

```bash
dart run build_runner watch --delete-conflicting-outputs
```

## Golden Rule: Where to Use getIt

| Location | Allowed? | Rationale |
|----------|----------|-----------|
| `presentation/` — inside `BlocProvider.create` | **Yes** | This is its primary purpose |
| `core/` — in `app_router.dart` | **Yes** | Instantiates BLoCs for specific routes |
| `domain/` — any file | **Never** | The domain must be pure, with no infrastructure |
| `data/` — datasources or repositories | **Never** | Dependencies arrive via constructor (injection) |
| Widgets (outside BlocProvider) | **Avoid** | Prefer receiving the BLoC via `context.read<>()` |

## Third-Party Modules

Dependencies that cannot be annotated directly (because they are from external packages) are registered via modules:

```dart
// lib/core/di/network_module.dart
@module
abstract class NetworkModule {
  @lazySingleton
  Dio get dio => Dio(BaseOptions(
    connectTimeout: const Duration(seconds: 15),
    receiveTimeout: const Duration(seconds: 30),
  ));
}

// lib/core/di/shared_preferences_module.dart
@module
abstract class SharedPreferencesModule {
  @preResolve  // resolves before init() completes (await)
  Future<SharedPreferences> get prefs => SharedPreferences.getInstance();
}
```
